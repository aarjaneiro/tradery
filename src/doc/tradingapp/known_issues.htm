Known Issues

  • Trying to close the Integrated Visual Trading Environment when there are multiple sessions open will cause a crash. Workaround: close each session individually and then close the application.

  • Editing a configuration's parameters in the Plugin Explorer bar if the configuration has its own UI and if the same configuration is also part of an open session may cause its parameters not to be reflected correctly in the session. Workaround: close the session(s) that contains the configuration to be edited, edit its parameters and re-open the session.

  • When opening some of the sample projects, an error message about missing source control information may be displayed. Workaround: this error message won't affect the project or the generated code in any way and it can be safely ignored.

  • Some Windows 98 users have reported this error when running TradingApp: "The file MISCPR.DLL is linked to missing output RPCRT4.DLL:UuidCreateSequential". Workaround: download and install the patch for this DLL by going to http://support.microsoft.com/?id=315575#XSLTH3133121123120121120120 .

• Name - can be any string; it will be listed in the plug-in tree.
  • Description - optional string that is used to describe the configuration
  • Id - a system generated predefined unique id identifying the configuration, read only
  • Enable saving signals - enables saving signals in real time if checked
  • Signal Files Path - the path under which the signal files will be saved
  • Base File Name - string that will prepended to the current date and time to create the signal file name
  • Extension - the extension of the signal files

This dialog only configures parameters related to the trading signal handling functionality. No configuration is needed for the statistics functionality of the FSHP - when the session is complete, statistics will automatically be calculated and displayed.

Multiple configurations of the FSHP can be created, each for a different path, base file name, extension.

Once created, a configuration's parameters are stored in the registry and the configuration will be available every time the TradingApp is run. A configuration can also be edited - all parameters except for its Id can be changed. If a configuration is no longer needed, it can be removed, which will delete it from the registry as well.

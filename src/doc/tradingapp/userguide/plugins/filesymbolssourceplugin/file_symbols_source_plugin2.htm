• Name - can be any string; it will be listed in the plugin tree.
  • Id - a UniquId value, generated internally, that can not be changed by the user
  • Description - optional string that is used to describe the configuration
  • Symbols list file name - the name of the file containing the list of symbols

Multiple configurations can be created, each for a different list of symbols.

Once created, a FSSP configuration's parameters are stored in the registry and will be available every time the TradingApp is run. A configuration can also be edited - all its parameters except for its Id can be changed. If a configuration is no longer needed, it can be removed, which will delete it in the registry as well.

The FSSP configurations correspond to static symbols list.

Symbols list file format

Symbols are stored in text files and can be separated by white space characters (blanc, tab or end of line) or by comma or semicolon. Lines starting with # or // are considered comments and ignored.

Here is an example of a symbols list file contents

msft yhoo; GOOG,
IBM
intc
// this is a comment
#this is another comment

TXN

 

Note - currently the symbols cannot contain whitespace characters (blanc, tab, new line). This will be fixed in a future release when symbols surrounded by quote characters will be accepted.

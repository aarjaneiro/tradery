• Name - can be any string; it will be listed in the plugin tree.
  • Description - optional string that is used to describe the configuration
  • Id - a predefined unique id identifying the configuration, read only
  • Data file path - the path under which the data files are located
  • Extension - the extension of the data files
  • Format - the format in which the data is stored in the data files

Multiple configurations can be created, each for a different set of symbols, data format, extension, path.

Once created, a configuration's parameters are stored in the registry and the configuration will be available every time the TradingApp is run. A configuration can also be edited - all parameters except for its Id can be changed. If a configuration is no longer needed, it can be removed, which will delete it from the registry as well.

The FD configurations correspond to static data sources, although the FD could be easily made dynamic, in which case the data files would have to be updated in real-time to contain the new data.

Data file formats

All data formats allow comment lines which start with $, # or //. These lines as well as all empty lines are ignored.

Each non comment line represents a bar of data. There are several data formats currently supported.
